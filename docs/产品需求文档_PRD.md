# 二手交易平台 (C2C Market) 产品需求文档 (PRD)

## 1. 产品背景与概述
本产品旨在打造一个类似“闲鱼”、“转转”的 C2C（个人对个人）闲置物品交易平台。系统分为面向普通用户的 **C端应用（移动端/H5/小程序）** 以及面向平台运营人员的 **B端后台管理系统（基于若依/RuoYi架构）**。
核心目标是提供一个安全、便捷、高效的二手物品流转环境，支持商品的发布、展示、交易、物流跟踪、即时通讯以及售后维权。

---

## 2. 系统模块设计

### 2.1 C端应用（用户侧）
根据当前数据库设计（`market_c2c`），C端系统划分为以下四大核心业务域：

#### 1) 用户域 (User Domain)
* **账号与基本信息**：支持邮箱（验证码）注册/登录，维护用户昵称、头像、密码等。
* **会员体系**：内置用户会员体系取代信用分。用户每消费10元可获得1点积分；系统通过用户的当前积分余额决定会员等级。退款时会扣除对应的当前积分余额，并且**会同步降级**以防止刷级行为。会员等级和对应的结算折扣在B端动态配置。
* **收货地址管理**：用户可添加多条收件地址（包含省市区、详细地址及收件人信息），并支持设置默认地址。
* **收藏夹功能**：用户可将感兴趣的商品加入收藏（Favorite），并在个人中心集中管理。

#### 2) 商品域 (Product Domain)
* **商品分类树**：展示平台维护的商品类目（支持多级父子分类及图标展示），方便用户通过类目筛选商品。
* **商品发布**：卖家（Seller）发布闲置物品，支持填写标题、图文描述、转让价、入手原价、成色（如9成新等），并上传多张商品图片（支持首图排序）。
* **商品浏览与互动**：买家浏览商品详情，系统记录商品的浏览量（Views）和点赞/想看数（Likes）。
* **商品状态管理**：卖家可控制商品状态（上架、下架、已售出）。

#### 3) 交易域 (Transaction Domain)
* **订单生成与支付**：买家对商品发起购买，绑定收货地址，生成唯一订单号（Order No），并记录订单总价和支付时间。
* **订单状态流转**：涵盖订单全生命周期（待付、待发、待收、完成、取消、退款中、已关闭）。
* **物流追踪**：卖家发货后录入快递公司及物流单号，买卖双方可查看物流状态更新。
* **售后与维权（退款）**：支持买家针对订单发起退款申请（录入金额及原因），卖家可同意或拒绝，状态包括：处理中、同意退款、拒绝退款、已取消。

#### 4) 互动域 (Interaction Domain)
* **私信与聊天**：买卖双方在交易前中后可通过站内信进行沟通（支持文本、图片消息）。
* **系统通知**：支持接收平台下发的系统通知及订单状态变更的自动提醒消息。消息具备已读/未读状态管理。

### 2.2 B端后台（运营管理侧）
结合现有的前端 `admin-frontend`（RuoYi-Vue框架），管理后台包含基础架构与市场业务两部分：

#### 1) 系统基础配置 (System Admin)
* **权限与组织架构**：用户管理、角色管理、菜单管理、部门管理、岗位管理等（若依原生功能）。
* **系统工具**：字典管理、参数配置、通知公告发布、操作日志监控等。

#### 2) 市场业务管控 (Market Business Admin)
* **C端用户管理**：查看全量注册用户，管控账号状态（如封禁违规用户），根据需要人工干预会员积分与等级。
* **会员等级配置**：管理员在后台配置各会员等级名称、积分门槛及对应折扣率（如黄金会员，500分，0.95折扣）。
* **商品审核与管理**：全量商品列表监控，支持按分类、状态检索。对违禁/违规商品进行强制下架操作。
* **分类类目维护**：管理 `mkt_category`，新增或调整分类名称、图标及排序权重。
* **订单与售后中心**：监控平台所有订单交易流水。在买卖双方出现退款纠纷时（如卖家拒绝退款），平台客服可介入处理（修改退款状态）。

## 3. 核心业务流程 (Core Workflows)

### 3.1 核心交易链路
### 3.1 核心交易链路
`商品浏览` -> `发起聊天(询价/问细节)` -> `提交订单(系统自动匹配会员等级计算折后价，买家选地址)` -> `付款` -> `卖家发货(填单号)` -> `买家签收(确认收货)` -> `交易完成(资金结算/按消费金额发放会员积分并重估等级)`。

### 3.2 售后维权链路
`买家申请退款` -> `卖家审核`：
- **卖家同意** -> `执行退款` -> `订单关闭`。
- **卖家拒绝** -> `买家申请平台介入` -> `后台管理员审核` -> `执行仲裁决定`。

## 4. 后端接口清单 (Backend API Endpoints)
为支撑上述前后端业务，需开发以下核心 RESTful API 接口（示例）：

### 4.1 App端接口 (面向C端用户)

* **用户与地址 (User & Address)**
* `POST /api/app/user/sendCode`: 发送邮箱验证码
* `POST /api/app/user/login`: 邮箱及验证码注册/登录
* `GET /api/app/user/profile`: 获取个人基本信息、当前积分及会员等级
* `GET /api/app/address/list`: 获取我的收货地址列表
* `POST /api/app/address/save`: 新增/修改收货地址
* `POST /api/app/favorite/toggle`: 收藏/取消收藏商品

**商品浏览与管理 (Product)**
* `GET /api/app/category/tree`: 获取分类树级联数据
* `GET /api/app/product/page`: 分页查询商品列表（支持搜索、分类过滤）
* `GET /api/app/product/{id}`: 获取商品详情及轮播图、卖家信息
* `POST /api/app/product/publish`: 发布闲置商品（包含主信息与多图关联）
* `PUT /api/app/product/status`: 卖家调整商品上下架状态

**交易与订单 (Order & Refund)**
* `POST /api/app/order/create`: 提交购买订单
* `POST /api/app/order/pay`: 模拟支付/支付回调接口
* `GET /api/app/order/my`: 获取我的订单列表（支持买家视角/卖家视角切换）
* `PUT /api/app/order/deliver`: 卖家发货，绑定物流信息
* `PUT /api/app/order/receive`: 买家确认收货
* `POST /api/app/refund/apply`: 买家提交退款维权申请

**消息与互动 (Message)**
* `GET /api/app/message/session`: 获取消息会话列表
* `GET /api/app/message/history`: 获取与某人的聊天记录
* `POST /api/app/message/send`: 发送聊天消息
* `PUT /api/app/message/read`: 标记消息已读

### 4.2 Admin端接口 (面向B端管理)

*(除了RuoYi自带的 `/system/*` 接口外，需新增以下业务接口)*

* `GET /api/admin/market/user/page`: C端用户列表分页查询及冻结操作
* `GET /api/admin/market/category/list`: 分类列表维护（增删改查）
* `GET /api/admin/market/product/page`: 全量商品列表，支持强行下架接口
* `GET /api/admin/market/order/page`: 平台订单大盘数据查询
* `PUT /api/admin/market/refund/intervene`: 平台客服介入处理退款申请的审批接口

## 5. 技术栈与系统架构 (Technology Stack & Architecture)

本项目采用前后端分离的现代化架构体系，充分考虑了多端适配与快速开发的诉求：

### 5.1 后端技术栈 (Backend)
- **核心框架**：Spring Boot (基于 RuoYi-Vue 基础架构)
- **安全与权限**：Spring Security + JWT 认证
- **ORM 与数据访问**：MyBatis / MyBatis-Plus
- **主数据库**：MySQL 8.x (存储用户、商品、订单等核心持久化数据)
- **缓存与中间件**：Redis (用于高频数据查询、验证码缓存、JWT Token 会话状态管理与过期时间控制等)
- **第三方集成**：Resend API (用于处理邮件及验证码下发)

### 5.2 前端技术栈 (Frontend - B端管理后台)
- **核心框架**：Vue 3.x (基于 Composition API)
- **构建工具**：Vite 6.x
- **UI 组件库**：Element Plus
- **状态管理**：Pinia
- **路由控制**：Vue Router 4.x
- **网络请求**：Axios

### 5.3 移动端技术栈 (Frontend - C端应用)
- **跨平台框架**：uni-app (支持一套代码编译为 H5、微信小程序、iOS/Android App)
- **核心语法**：Vue 3
- **样式预处理**：Sass/SCSS
